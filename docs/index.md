# ebhelper Provider

The `ebhelper` provider bridges the gap between AWS Elastic Beanstalk and Terraform by:

1. **Discovering** EB environment infrastructure (ASG, target groups, load balancers) at apply time — solving the chicken-and-egg problem where data sources fail on first apply
2. **Patching** ASG and ALB properties that have no native standalone Terraform resource

## When to Use This Provider

Use `ebhelper` when you need to:
- Reference the ASG name created by EB in other Terraform resources (like `aws_autoscaling_schedule`)
- Set ASG health check type to ELB (not exposed as an EB option setting in a usable way)
- Configure instance maintenance policies on EB-managed ASGs
- Set termination policies, cooldown, max instance lifetime, or warmup on EB-managed ASGs
- Modify ALB attributes on EB-created load balancers

For settings that EB exposes through option settings or that have native standalone resources (`aws_autoscaling_schedule`, `aws_autoscaling_policy`, etc.), use those directly.

## Example Usage

```terraform
provider "ebhelper" {
  region = "ap-southeast-2"
}

resource "ebhelper_environment_info" "app" {
  application_name = "my-app"
  environment_name = "my-app-production"
}

resource "ebhelper_asg_health_check" "app" {
  asg_name          = ebhelper_environment_info.app.asg_name
  health_check_type = "ELB"
}

resource "aws_autoscaling_schedule" "scale_up" {
  autoscaling_group_name = ebhelper_environment_info.app.asg_name
  scheduled_action_name  = "scale-up"
  recurrence             = "0 7 * * MON-FRI"
  desired_capacity       = 2
  min_size               = 2
  max_size               = 4
  time_zone              = "Pacific/Auckland"
}
```

## Authentication

The provider uses the standard AWS credential chain. Configure credentials via:
- Environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SESSION_TOKEN`)
- Shared credentials file (`~/.aws/credentials`)
- IAM instance profile

For cross-account access, use the `assume_role` block.
